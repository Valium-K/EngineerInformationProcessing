데이터베이스 설계 시 고려사항
	회복: 장애시 복구가능 해야함
	보안: 불법적인 노출이나 변경에 대비
	효율성: 최적화
	DB 확장: 운영에 영향을 주지 않으면서 데이터 추가 가능해야함
	
데이터베이스 설계순서
	1. 요구조건 분석
		* 용도 파악 후 명세작성
	2. 개념적 설계(정보 모델링, 개념화)
		* 현실세계를 추상적 개념으로 표현
		* 개념 스키마 모델링과 트랜잭션 모델링을 병행 수행
		* E-R 다이어그램 사용
	3. 논리적 설계(데이터 모델링)
		* 컴퓨터가 이해하는 논리적 자료구조로 변환
		* 개념 스키마를 평가 및 정제
		* 트랜잭션의 인터페이스 설계
		* 테이블 설계단계
	4. 물리적 설계(데이터 구조화)
		* 물리적 구조의 데이터로 변환
			* DB의 기본저장 단위는 저장 레코드이다.
		* 저장 구조 및 액세스 경로설정
	5. 구현

데이터 모델 구성요소: 개체, 속성, 관계
데이터 모델에 표시할 요소: 구조, 연산, 제약조건 (구제연)

E-R 다이어그램
	개체: 사각형
	속성: 원
	관계: 마름모
		* E-R 다이어그램의 관계(1, N) 는 Entity, Atribute 순서이다.
	
관계형 데이터베이스
	튜플(가로, 행)의 수를 카디널리티, 기수, 대응수 라고한다.
	속성(세로, 열)의 수를 디그리, 차수라고 한다.

후보키: 유일성과 최소성을 만족하는 기본키의 후보군
기본키: 후보키에서 선정된 기본키
대체키: 후보키에서 기본키를 제외한 나머지 키
슈퍼키: 유일성은 만족하지만 최소성을 만족하지 못하는 키

개체 무결성: 기본키를 구성하는 속성은 Null이나 중복이 있으면 안됨
도메인 무결성: 도메인 값은 도메인 범위 안에 속한 값이여야함
참조 무결성: 외래키 값은 Null 이거나 기본키의 값과 동일해야함

관계대수: 정보검색의 절차적 언어
	순수 관계 연산자: select(σ), Project(π), Join(⋈), Division(÷)
	일반 집합 연산자: 합, 교, 차, 교차곱 집합
	
관계해석: 정보검색의 비절차적 언어

정규화: 논리적 단계에서 수행
정규화 과정
	도부이결다조(두부이걸다조)
		* 부,이,결,다 제거
		* 조인 종속석 이용

반정규화 방법 종류
	1:1, 1:N, 슈퍼/서프타입 통합
중복 테이블 추가방법
	진행, 집계 특정부분 테이블 추가
	
데이터 사전(시스템 카탈로그): DB의 데이터에 대한 정의나 명세를 관리하는 시스템
	* 일반 사용자는 검색만 가능
	* 시스템이 자동으로 관리
	* 데이터 디렉토리는 시스템만 접근 가능

테이블 = 개체
칼럼 = 속성

CRUD
	우선순위: C>D>U>R
	검토사항: 아무것도 적히지 않은 행(테이블), C나 R이 없는 열(프로세스)을 확인

비트맵 인덱스
	분포도가 좋은(분포도가 낮은) 칼럼에 적합

뷰
	기본키를 포함한 뷰만이 삽입, 삭제, 갱신이 가능하다.
	논리적 독립성을 제공한다.
	독립적인 인덱스를 가질 수 없다.

파티션 설계
	사용시 성능 향상 but 조인시 비용증가
	범위 분할(일, 월 등으로 분할), 해시 분할(데이터가 고르게 분포시), 조합 분할(파티션이 클 경우)이 있다

분산 데이터베이스 설계
	테이블 위치 분산, 분할, 할당 이있다.
		* 분분할 로 외우자 or 산할당
		
분산 데이터베이스의 목표
	위치 투명성: 실제위치 몰라도 됨
	중복 투명성: 동일 데이터가 여러곳에 있더라도 하나인 것 처럼 사용
	병행 투명성: 병행 작업 해도 잘 돼야함
	장애 투명성: 장애 발생시에도 잘 돼야함
		* 위장중병 으로 외움

접근통제 종류
	DAC: 임의 - 만든사람이 정함
	MAC: 강제 - 시스템이 정함
	RBAC: 역할 - 직책에 따라 정함
	
접근통제 3요소: 정책, 보안모델, 매커니즘 (정보커)

접근통제에서의 기밀성, 무결성
	기밀성: 인가된 사용자만 접근 가능
	무결성: 인가된 사용자만 수정가능

논리백업: DB 객체 백업
물리백업: DB 파일 백업

DDL: create, alter, drop
DML: 
	select ... from ... where
	insert ... into ... values
	delete ... from ... where
	update ... set ... where
	* <>: 같지않다
	* _: 문자하나, %: 모든문자
DCL: commit, rollback, grant, revoke
	
	* define 절대없음
	
Left Outer Join: Inner Join 후 왼쪽 조인되지 않는 나머지 릴레이션에 Null 값을 넣어 반환


프로시저 필수구성
Declare(create Procedure 이름())
Begin
	...
end;
	* 실행은 execute, exec, call 로 한다
	* 제거시 drop procedure 이름
	* 일일 마감 같은 주기적인곳에 사용
	* 반환값 없어가 1개 이상
	
	
트리거 필수구성
Declare(create triger 이름 [before|after] on ...)
event
begin
	...
end;
	* DCL 사용 불가
	* 무결성, 오류 메시지 출력 등에 사용


사용자 정의 함수
Declare(create function 이름())
begin
	...
	return
end;
	* DML 중 select로 조회만 가능함
	* 말 그대로 함수만드는 것
	* DB에 포함돼어 DML에 함께작동
	* return 필수
	
프로시저, 트리거, 사용자 정의함수 삭제는 drop

웹서버: 페이지, 자료 등의 처리를 담당
WAS: DBMS와 통신하여 로직처리
	* [웹 서버 - WAS]를 합쳐 웹 응용 시스템이라 한다

ODBC: 마이크로 소프트에서 출시
MyBatis: SQL 문장을 분리해 XML로 저장하고 Mapping을 통해 SQL을 실행
