나선형 모델의 4가지 주요 활동순서
	계획 수립 - 위험 분석 - 개발 및 검증 - 고객평가
	
스크럼 개발 과정 진행 순서
	스프린트 계획 회의 - 스프린트 - 일일 스크럼 회의 - 스프린트 검토회의 - 스프린트 회고

XP의 5가지 핵심 가치
	의사소통, 단순성, 용기, 존중, 피드백

요구사항 개발 프로세스 순서 및 특징
	도출: 요구사항 수집
		* 인터뷰, 프로토타이핑, 유스케이스 등 사용
	분석: 사용자 요구사항 필터링
		* DFD, CASE, 자료사전 등 사용
	명세: 모델 작성 및 문서화
		* 소단위 명세서 사용 가능
	확인: 명세서가 잘 작성 되었는지 검토


DFD의 구성요소
	프로세스, 자료 흐름, 자료 저장소, 단말

CASE(요구사항 분석 자동화 도구)의 종류
	SADT: SoftTech사 에서 개발
	SREM(RSL/REVS): TRW사 에서 개발
	PSL/PSA: 미시간 대학에서 개발
	TAGS: 시스템 공학 방법 응용에 대한 자동 접근 방법

HIPO Chart의 종류
	시스템 분석 및 설계나 문서화에 사용
	
	가시적 도표(Visual Table of Contents): 시스템의 전체 구성을 게층(Tree) 구조로 표현
	총체적 도표(Overview Diagram): 프로그램의 기능을 기술한 도표
	세부적 다이어그램(Detail Diagram): 총체적 도표의 기능을 상세히 기술하는 도표
	
UML
	객체지향 모델링 언어
	구성요소: 사물, 관계, 다이어그램

럼바우 객체지향 분석기법
	구조적 기법: 객체 다이어그램
	행위적 기법: 상태 아이어그램

유스케이스 다이어그램의 구성 요소(행위)
	시스템, 액터, 유스케이스, 관계

시퀀스 다이어그램: (행위) 객체들이 주고받는 메시지의 관계를 표현
배치(Deployment) 다이어그램: (구조적) 프로세스, 컴포넌트 등의 물리적 위치를 표현
컴포넌트 다이어그램: (구조적) 컴포넌드간 관계를 표현

사용자 인터페이스의 기본 원칙
	직관성, 유효성, 학습성, 유연성


이미지 맵: 그림에 하이퍼링크를 연결
사이트 맵: 사이트의 전체 구조를 트리형태로 만든것

와이어프레임: 기획 단계 때 전체적 뼈대를 설계
	* 손그림, 일러스트, 포토샵 등 사용

스토리보드: 와이어프레임에 콘텐츠 설명, 이동 흐름 등을 추가한 문서
	* 개발자와 디자이너가 최종적으로 참고하는 작업 지침서
	* 파워포인트 등을 사용

품질 요구사항
	ISO/IEC 9126: 소프트웨어 평가 표준
	ISO/IEC 12119: 9126에 테스트 절차를 포함한 규정
	ISO/IEC 25010: 9126을 개정한 소프트웨서 국제 표준 (기능의 보안성을 따로 뺌, 성능 효율성을 추가)
	
	기능성: 요구사항을 정확하게 만족하는 기능
	사용성: 향후 다시 사용하고 싶은 정도

소프트웨어 아키텍처의 기본 원리
	모듈화, 추상화, 단계적 분해, 정보 은닉
	
소프트웨어 아키텍처의 설계
	상위 설계: 전체적 구조설계
		* 아키텍처 설계, 예비 설계 등
	하위 설계: 전체적 구조의 내부구조를 설계
		* 모듈설계, 상세 설계 등

객체지향
	연관성: 두 개 이상의 객체들이 상호 참조하는 관계

객체지향 분석의 방법론
	럼바우 기법: 객체(사이 관계) - 동적(제어, 흐름) - 기능(DFD사용) 모델 순서로 나누어 수행
	부치 방법: 미시적, 거시적 개발 프로세스를 모두 사용해 분석
	Jacobson 방법: 유스 케이스를 사용하는 분석
	Coad와 Yourdon 방법: E-R 다이어그램을 사용해 분석
	Wirfs-Brock 방법: 분석과 설계간 구분 없이 고객 명세서를 평가하는 분석

객체지향 설계 원칙
	개방-폐쇄 원칙: 기존코드를 병견하지 않고 기능 추가해야함
	리스코프 치환 원칙: 서브타입은 기반타입으로 교체 가능해야함
	인터페이스 분리 원칙: 자신이 사용하지 않는 인터페이스와는 관계 맺지 않아야함
	의존 역전 원칙: 추상성이 높은 클래스와 의존 관계를 맺어야함
	
모듈의 결합도
	(낮음)									  (높음)
	자료 - 스탬프 - 제어 - 외부 - 공통(공유) - 내용
	
	* 외부 결합도: 외부변수 참조
	* 공통(공유) 결합도: 공통 데이터 영역을 여러 모듈
	* 내용 결합도: 모듈이 다른 자료를 직접 참조

모듈의 응집도
	(강함)														(약함)
	기능적 - 순차적 - 교환(통신)적 - 절차적 - 시간적 - 논리적 - 우연적
	
	순차적: 모듈 출력을 바로 받아 입력으로 받음
	교환(통신)적 : 동일한 입력에 대한 여러 출력
	절차적: 모듈 내 모듈들의 순차적인 수행모습
	
팬인/팬아웃
	팬인: signal-in 즉, 화살표 받는 수
	팬아웃: signal-out 즉, 화살표 내보내는 수
	
공통모듈 명세기법
	정확성, 명확성, 완전성, 일관성, 추적성
	
공통모듈 재사용 규모 분류
	함수와 객체, 컴포넌트, 애플리케이션
	
코드의 종류
	순차 코드: 차례로 일련번호 부여
	블록 코드: 공통성을 블록으로 구분 후 일련번호 부여
	10진 코드: 도서 분류식
	그룹분류 코드: 일정 기준에 따라 구분 후 일련번호 부여
	연상 코드: 항목의 명칭을 줄여서 분자와 코드로 부여
	표의 숫자코드: 물리적 숫자를 그대로 코드에 부여
	합성 코드: 2개 이상의 코드를 사용
	
미들웨어의 종류
	DB: DB 접속 미들웨어
	RPC: 원격 프로시저를 로컬 처럼사용
	MOM: 메시지 기반의 비동기형 메시지를 전달
	TP-Monitor: 트랜잭션을 처리 및 감시하는 미들웨어
	ORB: 코바 표준 스펙을 구현한 객체 지향 미들웨어
	WAS: 동적 콘텐츠 처리
