	CBD (컴포넌트 기반 방법론)

		* 임베디드 소프트웨어를 만드는데 적합

소프트웨어 재사용
	합성중심: 모듈을 사용
	생성중심: 명세를 구체화

CASE(Computer Aided Software Enginerring)
	전체 개발 과정을 컴퓨터와 전용 툴을 이용해 자동화함
	* 생명주기의 모든 단계 연결 및 그래픽 지원등 특징.

소프트웨어 비용 결정
	프로젝트 요소: 복잡도, 시스템 크기, 신뢰도
	자원 요소: 인적 자원, 하드웨어 자원, 소프트웨어 자원
	생산성 요소: 개발자 능력, 개발 기간

비용 산정 기법
  하향식
	전문가 감정: 두 명 이상의 전문가가 주관적으로 선정
	델파이 기법: 전문가 감정의 보완으로 여러명의 전문가가 서로의 산정비용을 비교하며 조율
  상향식
	LOC(Source Line of Code): 원시 코드 라인 수
		노력(인월): 전체 개발 기간 * 전체 투입 인원
		개발비용: 노력(인월) * 개발자 1명의 월급
	
		* COCOMO 모형이다
		프로그램 규모에 따른 모형
		  조직형: 소규모. 약 5만 라인 이하의 사무, 과학용 프로그램
		  반-분리형: 중간규모. 약 30만 라인 이하의 컴파일러, 운영체제 등
		  내장형: 대형규모. 약 30만 라인 이상의 미사일 유도 시스템, 도시 신호 제어 시스템 등
	
		Putnam 모형: 생명 주기 예측 모형으로 Rayleigh-Norden 곡선 노력 분포도를 기초로 함	SLIM 이라는 툴이 있음
		기증 점수(FP) 모형: 소프트웨어 기능 증대 요인을 점수로 바꿔 산정	ESTIMACS 라는 툴이 있음

프로젝트 일정 계획
	PERT: 프로그램 평가 및 검토 기술
	CPM: 임계 경로 기법
		* 간선의 경우의 수 중 가장 큰 비용을 선택
		* 위 2개는 그래프 형식임
	간트차트: 시간선 차트를 이용.
		* 위는 차트(표)를 이용함.

소프트웨어 개발 표준
	ISO/IEC 12207: 소프트웨어 생명 주기의 표준
	CMMI: 능력 성숙도 통합 모델로 "개발조직"의 능력을 평가함
	SPICE: 소프트웨어 처리 개선 및 능력 평가로 "소프트웨어"를 평가함
	
M2M: 기계과 기계사이 통신
메시 네트워크: 라우터들을 기지국 처럼 사용하여 안정적인 네트워크를 사용 가능
Wi-Sun: 장거리 무선 통신을 위한 저적력 장거리 통신 기술
파장 분할 다중화(WDM): 광섬유를 이용한 통신기술
소프트웨어 정의 데이터 센터(SDDC): 인력의 개입 없이 소프트웨어 조작만으로 관리 및 제어되는 데이터 센터
개방형 링크드 데이터(LOD): 웹상에 공개된 연계 데이터
피코넷: 기계들 사이 블루투스로 통신함

802.11e: Qos 기능을 지원하는 802.11 버전

경로 제어 프로토콜
	IGP(Interior Gateway Protocol): 내부 게이트웨이 프로토콜
	  RIP
		내부 게이트웨이 프로토콜로 Bellman-Ford 알고리즘을 사용한다.
		소규모 동종 네트워크에서 효과적
		최대 홉 수를 15로 제한
	  OSPF
		경로 탐색을 Dijkstra 알고리즘을 사용
	EGP(Exterior Gateway Protocol): 게이트웨이 간의 라우팅에 사용
	BGP(Vorder Gateway Protocol): EGP의 단점을 보완. 첫 라우팅 테이블만 교환 후 변경되는 곳만 변경

매시업: 웹 에서 제공하는 서비스를 이용하여 새로운 서비스를 만드는 것

브로드 데이터: 기업 마케팅 데이터
메타 데이터: 데이터를 정의하고 설명해주는 데이터
타조: 하둡 기반의 분산 데이터 웨어하우스로 우리나라가 개발
OLAP: 통계적인 요약 정보를 분석하여 의사 결정에 활용하는 것
	* %-% 형식의 이름, Pivoting, Slicing, Dicing 연산이 있다.

교착상태 필요 충분 조건: 상호 배제, 점유와 대기, 비선점, 환형 대기. 이 4가지가 모두 갖춰지면 교착상태가남
교착상태의 해결 방법
	예방 기법: 교착상태의 4가지 조건 중 하나를 제거. 자원의 낭비가 심함
	회피 기법: 적절히 피해감. 은행원 알고리즘을 사용
	
시큐어 코딩: 소프트웨어 구현단계에서 보안요소를 고려하며 코딩. 개발 참여자에게 코딩 교육을 실시함

입력 데이터 취약점: SQL 삽입, XSS, 위험한 파일 업로드, 신뢰되지 않는 URL 자동연결, 메모리 버퍼 오버플로
보안 취약점: 부적절한 권한 인가, 취약한 암호화 알고리즘, 평문전송, 하드코드된 비밀번호
시간 및 상태 최약점: 동기화 오류,Flag 세우는 순간 다른 프로세서가 가로채는 경우, 교착상태, 무한 루프
코드 오류: 널 포인터 역참조(널 포인터 주소에 데이터를 쓰는 경우), 부적절한 자원해제, 해제된 자원 사용, 초기화되지 않은 변수 사용

접근지정자		(가능)
	public   : 클래스 내부, 패키지 내부, 하위 클래스, 패키지 외부
	protected: 클래스 내부, 패키지 내부, 하위 클래스
	default  : 클래스 내부, 패키지 내부
	private  : 클래스 내부
	
개인키 암호화 방식
	스트림 방식: LFSR, RC4	(LR 로 외운다)
	블록 암호화 방식: DES, SEED, AES, ARIA	(위 2개 빼고 다)
		* 공개키 방식보다 파일 크기가 작다.

해시: 알고리즘이다.

스머핑: 브로드 캐스트 수신지를 모두 한 호스트IP로 바꿔치기해 서비스 거부시킴
Land: 수신지, 송신지를 피해자IP로 바꿔치기해 패킷을 보내 자신에 대해 무기한 응답하게 시킴
백도어 방지법: 무결성, 로그 분석, SetUID파일 검사

행위기반 인증: 서명, 몸동작
소유기반 인증: 신분증, 토큰, OTP

침입탐지 시스템: 실시간으로 시스템 오남용을 탐지
침입방지 시스템: 방화벽+침입탐지 시스템으로 능종적으로 방어함
웹 방화벽: 일반 방화벽이 탐지 못 하는 SQL 삽입, XSS 등을 방어할 목적. 즉 웹 서버에 특화된 방화벽
NAC: 불법적인 소프트웨어 설치 방지, IP주소를 모니터링, 비인가 장치 탐지
ESM: 로그 및 이벤트를 관리
