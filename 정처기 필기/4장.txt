Batch Program 주기에 따른 분류
	정기 배치, 이벤트성 배치, On-Demand 배치

Batch Program 필수요소
	대용량 데이터, 자동화, 경고성, 안정성/신뢰성, 성능

	* 배치 스케줄러는 잡 스케줄러라고도 불린다

C언어 기억 클래스
	Static, auto, register, extren

연산자 우선순위
	[++, --] - [*, /, %] - [+, -] - [시프트 연산]
	* 증감이 1등, 꼴찌가 시프트연산, 그 사이에 사칙연산 순서라고 외우자

printf("%2.1f", 123.45)
	* 입력값이 정수부보다 크면 그냥 전부표시
	* 소수점 1자리에서 반올림을 한다
	* 답은 123.5
	
세그먼테이션
	프로그램을 함수 같은 논리적 단위로 나눈 단위
	궁극적 이유는 기억공간 절약
	외부 단편화 발생 가능
	
세그먼테이션 주소 변환 방법
	1. 가상주소 인덱스확인
	2. 가상주소의 크기가 세그먼트 맵 테이블에 한계번지보다 작거나같으면
	3. 해당 맵 테이블의 기준번지와 가상주소 크기를 더한 값을 반환
	4. 아니면 트랩
페이지 교체 알고리즘
	OPT: 최적교체
	FIFO: First In First Out
	LRU: Least Recently Used (가장 오랫동안 사용하지 않은)
	LFU: Least Frequently Used
	NUR: Not Used Recently
		* 참조비트, 변형비트 조합 순으로 교체우선순위가 낮아진다
	* FIFO 빼고는 다 3글자의 알고리즘

페이지 크기가 커지면 페이지 수가 적어지므로
	오버해드감소 but 단편화 확률 증가
워킹셋: 일정 시간동안 자주 참조하는 페이지들의 집합
스레싱: 페이지 교체 소요시간이 처리시간보다 길어짐

프로세스 스케줄링 기법
	선점 스케줄링: 우선순위 뺐어씀
	비-선점 스케줄링: 못 뺐고 기다림
	
프로세스 스케줄링 알고리즘: 무조건 표를 그려서 풀자
	FCFS: FIrst Come First Service
	SJF: Shortest Job First
		* 제출시간은 해당 프로세스 도착시간임
	HRN: Hightest Response-ratio Next
		* 실제 실행시간 / 전체 시간

서브네팅시에 네트워크부가 0부터 시작함에 유의


OSI 모델
	응용
	표현: 정보의 형식 변환
	세션: 동기점 체크
		* 위 3층은 Http, FTP 등의 프로토콜 이용
	전송: [게이트웨이]	End-to-End간 투명한, 균일한 데이터 전송	TCP, UDP
	네트워크부: [라우터]	데이터의 교환 및 중계, 경로설정(routing)	IP, ICMP, ARP
	데이터 링크: [브리지, 스위치, 랜카드]	신뢰성있고 효율적인 정보전송, 오류제어
	물리: [리피터]

프로토콜의 기본요소: 신텍스, 시멘틱스, 타이밍
